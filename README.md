# DevOps Lab: WordPress with Prometheus & Grafana Monitoring

Учебный проект (лабораторная работа) по развертыванию инфраструктуры для WordPress с настроенным стеком мониторинга и логирования на базе Docker Compose.

## 🚀 Архитектура стека

В проекте развернуты следующие контейнеры:
* **WordPress** — веб-сайт и движок блогов.
* **MySQL** — реляционная база данных для хранения данных WordPress.
* **Prometheus** — система сбора и хранения метрик.
* **Grafana** — платформа для визуализации метрик и построения дашбордов.
* **Node Exporter** — сборщик системных метрик хоста (процессор, память, диски Arch Linux).
* **cAdvisor** — сборщик метрик использования ресурсов на уровне каждого Docker-контейнера.
* **Redis** — используется cAdvisor в качестве бэкенда для корректного кэширования и хранения исторических данных метрик.
* **MySQL Exporter** — сборщик метрик производительности базы данных MySQL (метрики InnoDB, кэша, соединений и т.д.).

> ⚠️ **Важное замечание:** cAdvisor требует стабильного слоя хранения для корректной агрегации некоторых метрик контейнеров. Для решения этой проблемы в стек был успешно интегрирован контейнер Redis.

## 🏁 Быстрый старт

1. Склонируйте репозиторий (или скачайте файлы):
   ```bash
   git clone https://github.com/quirkyQuack6/devops-lab.git
   cd devops-lab
   ```

2. Запустите все сервисы в фоновом режиме:
   ```bash
   docker compose up -d
   ```

3. Проверьте статус запущенных контейнеров:
   ```bash
   docker compose ps
   ```

## 📊 Мониторинг и метрики

### 1. Сбор метрик (Prometheus)
Prometheus настроен на автоматический сбор (scrape) метрик из двух источников:
* Node Exporter: `http://node-exporter:9100/metrics`
* cAdvisor: `http://cadvisor:8080/metrics`

### 2. Визуализация (Grafana)
Для отображения метрик в Grafana рекомендуется импортировать следующие популярные дашборды:
* **Для хоста (Node Exporter):** Дашборд ID `1860` (Node Exporter Full).
* **Для контейнеров (cAdvisor):** Дашборд ID `14282` (Docker Containers Dashboard).
* **Для базы данных (MySQL Exporter):** Дашборд ID `7362` (MySQL Overview) — профессиональный дашборд для отслеживания соединений, трафика, работы InnoDB и производительности запросов.

### 3. Безопасность и сбор метрик базы данных (MySQL Exporter)
Для обеспечения безопасности проекта сбор метрик с базы данных выполняется не под учетной записью `root`. В MySQL был создан отдельный выделенный пользователь `exporter` с ограниченными правами (только на чтение системных таблиц производительности).

Команды, которые использовались для создания пользователя внутри MySQL:
```sql
CREATE USER 'exporter'@'%' IDENTIFIED BY 'ваш_пароль_экспортера';
GRANT PROCESS, REPLICATION CLIENT, SELECT ON *.* TO 'exporter'@'%';
FLUSH PRIVILEGES;
```
*Контейнер `mysql-exporter` авторизуется в базе данных, используя эти учетные данные через переменную `DATA_SOURCE_NAME`.*

## 🛑 Остановка проекта

Чтобы остановить все контейнеры и сохранить данные в volumes:
```bash
docker compose down
```

Если вы хотите полностью удалить контейнеры, сети и volumes (очистить данные):
```bash
docker compose down -v
```
